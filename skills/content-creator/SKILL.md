# content-creator

## Description

内容创作助手 - 帮助生成技术文档、视频脚本、产品文案等内容，直接产生经济效益。支持项目文档、技术博客、营销文案、视频脚本等多种内容形式的创作。

## When to Use

当用户需要创建以下内容时激活此技能：
- 技术文档（API文档、开发指南、架构文档）
- 项目文档（README、项目介绍、使用手册）
- 视频脚本（教程视频、产品演示、技术分享）
- 营销文案（产品描述、推广文案、广告语）
- 博客文章（技术博客、行业分析、经验分享）
- 演讲稿（技术演讲、产品发布、会议发言）

## Tools

此技能需要使用以下工具：
- `Read` - 读取现有文档和参考材料
- `Write` - 创建新文档
- `SearchReplace` - 修改现有内容
- `SearchCodebase` - 搜索代码库获取技术信息

## Workflow

1. **理解需求** - 明确内容类型、目标受众、风格要求
2. **收集信息** - 读取相关文件，搜索代码库，了解项目背景
3. **创作内容** - 根据需求生成高质量内容
4. **保存文件** - 将内容保存到合适的位置
5. **更新记忆** - 记录创作内容和收益信息

## Content Types

### 技术文档
- API 文档：接口定义、参数说明、示例代码
- 开发指南：环境搭建、快速开始、最佳实践
- 架构文档：系统架构、技术选型、设计决策

### 视频脚本
- 教程视频：步骤讲解、操作演示、要点总结
- 产品演示：功能介绍、使用场景、价值主张
- 技术分享：概念讲解、案例分析、经验总结

### 营销文案
- 产品描述：功能特点、优势亮点、使用场景
- 推广文案：吸引人的标题、核心卖点、行动号召
- 广告语：简洁有力、易于记忆、传播性强

## Best Practices

1. **了解受众** - 根据目标读者调整内容深度和风格
2. **结构清晰** - 使用标题、列表、代码块等增强可读性
3. **实例丰富** - 提供具体的代码示例和操作步骤
4. **语言简洁** - 避免冗余，直击要点
5. **价值导向** - 突出内容对读者的实际价值

## Examples

### 示例1：生成项目README

用户：帮我为coffeeclaw项目生成一个专业的README文档

思考：
1. 需要读取项目结构和关键文件了解项目
2. 生成标准的README结构
3. 包含项目介绍、功能特性、安装使用、贡献指南等

执行：
- 读取 package.json 了解项目信息
- 读取 src/ 目录了解代码结构
- 生成 README.md 文件

### 示例2：创作视频脚本

用户：我要做一个5分钟的CoffeeClaw功能介绍视频，帮我写脚本

思考：
1. 5分钟约750-1000字
2. 结构：开场(30s) → 功能介绍(3min) → 使用演示(1min) → 结尾(30s)
3. 语言要口语化，适合视频讲解

执行：
- 了解CoffeeClaw核心功能
- 创作分镜脚本
- 保存到 documents/video-scripts/

## Output Location

创作的内容默认保存到：
- 文档：`~/.openclaw/workspace/coffeeclaw/documents/`
- 脚本：`~/.openclaw/workspace/coffeeclaw/documents/scripts/`
- 文案：`~/.openclaw/workspace/coffeeclaw/documents/copy/`

## Integration with Memory

创作完成后，自动更新以下记忆文件：
- `PROGRESS.md` - 记录创作进度
- `DECISIONS.md` - 记录创作决策
- 更新经济效益追踪数据
